## SLIDE 2: THE PROBLEM

**Headline**: [Problem in 5 words]

**Visuals**:
- Customer pain point illustration
- Market size number (big)
- Underserved segment callout

**Text**:
- 2-3 sentences describing the problem
- Focus on: WHO suffers, WHY they suffer, SCALE of suffering

**Example**:

```
THE PROBLEM
──────────

SMEs waste 40% of AI budget on:
  • Repeated context setup (GPT forgets you)
  • Switching between tools (email, Slack, ChatGPT)
  • Managing API costs (OpenAI enterprise pricing)

→ R$50B market, 70% adopt (by 2025)
→ But only 15% use AI effectively

Opportunity: Cost-effective AI + memory for SMEs
```

**Speaker Notes**:
- "What's broken today?"
- Tell a short story (customer example)
- Show the cost/time/pain quantified
- Don't mention your solution yet

## SLIDE 5: PRODUCT DEMO (Live or Video)

**Format options**:

### Option A: Live Demo (2-3 min)
- Live system showing:
  1. Login screen
  2. Send a message
  3. Show conversation memory being used
  4. Show memory improving response
- **Risk**: Something might break. Have backup screenshot.

### Option B: Video Demo (2 min)
- Recorded walkthrough
- Edit smoothly with transitions
- Show happy path (no errors)
- Include customer testimonial at end

### Option C: Screenshot + Narration
- Key screenshots on slide
- You narrate what each does
- Simplest, safest option

**Script Example**:

```
"Let me show you what [Company] does.

[Open dashboard]
This is where you access the AI. Type a question.

[Send message]
The AI responds instantly with context from your 
previous conversations.

[Highlight memory]
See here? ALICI remembered that you work in fintech 
and uses domain-specific knowledge. ChatGPT would 
start from scratch.

[Next message]
Now we upgrade to Pro plan.
[Show upgrade]
30 seconds, and you get advanced features: API access, 
analytics, custom models.

That's it."
```

**Speaker Notes**:
- Keep it simple and fast
- Wow moment: Show memory in action
- Don't get bogged in technical details
- Have a Plan B if tech fails

## TIPS FOR PITCHING THIS DECK

1. **20-minute pitch rule**: 20 min presentation + 10 min Q&A
   - Slide 1-12: ~90 seconds each
   - Let good questions extend discussion

2. **Tone**: Confident, not arrogant. Data-driven, not spammy.
   - "We have validation X" (not "We will dominate")
   - Show traction early (slide 7)

3. **Personalization**: Customize for your audience
   - Talk to VC focused on AI? Add tech depth.
   - Talk to CVC at large enterprise? Add enterprise GTM.

4. **Story arc**: Problem → Solution → Team → Traction → Ask
   - Start hooks: Start with compelling problem
   - Build momentum: Traction proves you can execute
   - Close strong: Vision + clear ask

5. **Practice**:
   - Present 10+ times before investor pitches
   - Get feedback, refine
   - Know numbers cold (no stumbling)
   - Be ready to pivot: Have backup slides ready
