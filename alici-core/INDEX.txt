📂 DESCRIÇÃO DOS ARQUIVOS


🔌 DATABASE

  database/neon.py
  └─ Classe NeonDB para gerenciar Neon PostgreSQL
    ├─ Conexão automática ao Neon
    ├─ Criação de tabelas
    ├─ Logging de métricas
    ├─ Versionamento de modelos
    └─ Rastreabilidade de predições
  
  Usar: from database.neon import db


🧠 MODELOS

  models/image_branch.py
  └─ Ramo CNN para processamento de imagens
    ├─ Input: (32, 32, 3) - RGB
    ├─ Arquitetura: Conv2D → MaxPool → Conv2D → Dense(128)
    └─ Output: (128) features
  
  models/text_branch.py
  └─ Ramo LSTM para processamento de texto
    ├─ Input: (50,) - Sequência tokenizada
    ├─ Arquitetura: Embedding → LSTM → LSTM → Dense(128)
    └─ Output: (128) features
  
  models/audio_branch.py
  └─ Ramo Dense para processamento de áudio
    ├─ Input: (13,) - MFCC features
    ├─ Arquitetura: Dense → Dense → Dense → Dense(128)
    └─ Output: (128) features
  
  models/multimodal_model.py
  └─ Fusão dos 3 ramos
    ├─ Concatenate (384 features)
    ├─ Dense + Dropout + BatchNorm
    └─ Output: (256) classes


🎓 TREINAMENTO

  training/trainer.py
  └─ Classe TrainerMultimodal
    ├─ compilar_modelo() - Adam com LR=1e-4
    ├─ treinar() - Com callbacks automáticos
    ├─ avaliar() - Validação
    ├─ fazer_predicao() - Inferência
    ├─ salvar_modelo() - Salva + registra no Neon
    ├─ carregar_modelo() - Carrega pré-treinado
    └─ NeonLoggingCallback - Logging automático


⚡ API

  api/main.py
  └─ FastAPI para Render
    ├─ GET / - Status
    ├─ GET /health - Health check
    ├─ GET /status - Status detalhado
    ├─ POST /predict - Inferência
    ├─ GET /logs/treino - Histórico
    ├─ GET /modelos - Lista de versões
    ├─ POST /reload-model - Recarregar
    └─ GET /docs - Swagger

🎯 FLUXOS COMUNS


FLUXO 1: Treinar & Deploy
  1. Colab com colab_finetuning.py (GPU training)
  2. Download alici_core.h5
  3. Upload para GitHub
  4. Deploy em Render (auto)
  5. Use /predict endpoint


FLUXO 2: Desenvolvimento Local
  1. Setup: cd alici-core && pip install -r ../requirements.txt
  2. Test: python teste_completo.py
  3. Dev: Edite código
  4. Run API: cd api && uvicorn main:app --reload
  5. Test: curl http://localhost:8000/docs


FLUXO 3: Production Maintenance
  1. Treinar novo modelo no Colab
  2. Register no Neon (automático)
  3. Upload .h5 para GitHub
  4. Render pull & restart
  5. Monitor via /logs/treino

📋 CHECKLIST PRÉ-USO

Antes de começar:
  ☐ Clone/download alici-core
  ☐ Tenha Python 3.11+ instalado
  ☐ Tenha Neon account (opcional, funciona sem)
  ☐ Tenha Render account (para deploy)

Antes de treinar:
  ☐ Prepare dados (imagens 32x32, textos, áudios WAV)
  ☐ Normalize dados
  ☐ Configure DATABASE_URL (se usar Neon)

Antes de fazer deploy:
  ☐ Treine um modelo
  ☐ Salve como alici_core.h5
  ☐ Update requirements.txt
  ☐ Configure env vars no Render
  ☐ Git push
